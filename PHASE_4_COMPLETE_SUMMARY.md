## üéì DESIGN SYSTEM PRINCIPLES

### 1. Semantic Over Descriptive
- ‚úÖ **Good:** `variant="primary"`, `className="text-heading-1"`
- ‚ùå **Bad:** `variant="goldButton"`, `className="text-5xl-serif-black"`

**Why:** Semantic names describe purpose, making code self-documenting

### 2. Consistency Over Flexibility
- ‚úÖ **Good:** Use `py-section-md` everywhere
- ‚ùå **Bad:** Mix `py-20`, `py-24`, `py-section-md`

**Why:** Consistency creates predictable, maintainable systems

### 3. Variants Over Inline Styles
- ‚úÖ **Good:** `<Card variant="elevated">`
- ‚ùå **Bad:** `<Card className="bg-white/[0.02] border-white/10 hover:border-white/20">`

**Why:** Variants are reusable, testable, and maintainable

### 4. Progressive Enhancement
- ‚úÖ **Good:** Keep backward compatibility with deprecated variants
- ‚ùå **Bad:** Break existing code with immediate removals

**Why:** Gradual migration reduces risk and downtime
